# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.2.x   | ✅        |
| < 1.2   | ❌        |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email: security@thefiredev.com

Phantom handles sensitive client data and operates agent identities. We treat security reports with the highest priority.

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if you have one)

We will acknowledge receipt within 24 hours and provide a resolution timeline within 5 days.

## Scope

In scope:
- Agent identity isolation
- Client data access controls
- Authentication and authorization
- Voice/email impersonation safeguards
- API endpoint security

Out of scope:
- Third-party platform vulnerabilities
- Social engineering of human operators
- Physical security

## Disclosure Policy

We follow coordinated disclosure. Given the sensitivity of Phantom's operations, we ask for 90 days to resolve before public disclosure.
